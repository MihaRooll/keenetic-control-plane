# verify-loop

> Паттерн: правка → проверка → фиксация. Связано с [`docs/cursor-agent-best-practices.md`](../docs/cursor-agent-best-practices.md).

## For agents

**Когда:** после серии edit; перед «готово» / commit / PR; баг с repro.

**Применяй:**
1. Сделай изменение
2. Запусти релевантную проверку проекта (typecheck / test / lint / smoke)
3. Если fail → fix → повтори
4. Для багов: детальный repro; при необходимости Debug Mode
5. Сообщи: что прогнал + результат (не «должно работать»)

**Не делай:** объявлять done без проверки; гонять весь suite если есть узкий таргет; игнорировать красный вывод.

---

## Что гонять

| Изменение | Минимум |
|-----------|---------|
| TS/JS логика | typecheck + затронутые tests |
| Скрипт / hook | parse/smoke из docs проекта |
| Docs only | ссылки/индекс; код-тесты не обязательны |
| UI | ручной/browser check по задаче |

Подставь команды из `AGENTS.md` / package scripts проекта.

---

## Чеклист перед done

- [ ] Команда(ы) проверки выбраны по типу изменения
- [ ] Exit 0 / ожидаемый артефакт
- [ ] Failures исправлены или явно эскалированы
- [ ] Краткий отчёт пользователю: что проверено
